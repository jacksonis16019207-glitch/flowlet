# Deliverables Checklist

## Use This Reference

作業の締めで、何を更新すべきか迷ったときだけ参照する。毎回全部を作るのではなく、今回のタスクに必要なものだけを選ぶ。

## Candidate Deliverables

- 実装コード
- テストコード
- API 仕様メモ
- DB 設計メモ
- 画面仕様メモ
- セットアップ手順
- 開発コマンド追加の説明
- 運用影響の説明
- 記事候補メモ
- 記事アウトラインまたはドラフト
- Skill 改善提案

## Completion Check

- 要件と完了条件に対して不足がないか
- 仕様変更があるのに `docs/` 更新を漏らしていないか
- 実装したのに検証結果を示していない項目がないか
- 記事化や Skill 化の価値がある学びを見落としていないか
- コミット対象に不要差分が混ざっていないか
