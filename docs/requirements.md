# flowlet 要件定義

## 背景

ユーザーは複数の銀行口座を使い分けて生活費、カード引き落とし、投資、貯金を管理している。
以前は SBIネット銀行 の目的別口座を使って用途ごとに残高を分けていたが、解約後に SBI新生銀行 のハイパー預金へ資金を集約したことで、目的別残高の見通しが失われた。
また、各種カードや支払いの引き落とし額に対して三菱UFJ銀行のメイン口座残高が足りているかを都度電卓で確認しており、手間が大きい。

## 何に困っているか

- 1つの実口座にまとまった資金を、目的別残高として把握しづらい
- 現在の残高を把握するために、入出金や振替を頭の中や手計算で追う必要がある
- カード引き落とし予定額に対して、メイン口座残高が足りるかの確認が面倒
- 過去の記録を残したいが、まずは 2025-12-25 の給料日以降を起点にしたい

## 作りたいアプリ概要

`flowlet` は、実在する銀行口座とアプリ内の目的別口座を分けて管理し、資金の配置状況と引き落とし余力を見える化する家計・資産管理アプリである。

実口座の残高と取引履歴を管理しつつ、アプリ上では目的別の仮想口座を持てるようにする。これにより、銀行側で目的別口座がなくても、用途別の残高を維持して把握できる。

## 対象ユーザー

- 主ユーザーはアプリ作成者本人
- 個人利用を前提とし、当面はマルチユーザー対応を行わない

## 現在の口座構成

### 三菱UFJ銀行

- メイン口座
  - 給与受取
  - PayPayカード以外のカード引き落とし
  - 日常の主な支払い口座
- 使い分け口座
  - 趣味用
  - 健康用
  - 給料日にメイン口座から一定額を振替
  - 利用時はメイン口座へ戻す

### PayPay銀行

- PayPayカードの引き落とし口座
- 三菱UFJ銀行から PayPay 残高へチャージし、PayPay 残高から送金する運用がある

### SBI新生銀行

- 普通預金
- ハイパー預金
- 投資用、貯金用の資金を保持
- 給料日に一定額を普通預金へ入れ、ハイパー預金へ振替
- 利用時はハイパー預金から普通預金へ戻し、三菱UFJ銀行メイン口座へ振込

### 旧 SBIネット銀行 の管理単位

- 目的別口座
  - 親孝行
  - 定期支払
  - 特別貯金
  - 衣服・美容
- ハイブリッド預金
  - 投資用

## 解決したい課題

### 課題1: 目的別残高の見える化

実口座は1つでも、アプリ上では複数の目的別口座を持ち、用途ごとの残高を常に確認できるようにしたい。

### 課題2: 現在残高の再現

入出金、振替、口座間移動を記録し、現在の実口座残高と目的別残高を再現できるようにしたい。

### 課題3: 引き落とし不足の事前確認

カードや定期支払いの引き落とし予定額を登録し、特定日時点で対象口座の残高が足りるかを事前に確認したい。

## 主要ユースケース

### ユースケース1: 目的別口座で残高を管理する

- ユーザーは実口座を登録する
- ユーザーは目的別口座を作成する
- ユーザーは実口座内の資金を目的別口座へ配分する
- ユーザーは目的別の残高と未配分残高を確認する

### ユースケース2: 取引を記録して残高を更新する

- ユーザーは入金、出金、振替、口座間移動を記録する
- ユーザーは必要に応じて目的別口座とのひも付けを設定する
- アプリは残高を更新する

### ユースケース3: 引き落とし余力を確認する

- ユーザーはカードや固定費の引き落とし予定を手入力で登録する
- アプリは対象口座ごとの引き落とし予定合計を算出する
- アプリは残高不足の有無と不足見込み額を表示する

### ユースケース4: 過去明細を読み込む

- ユーザーは CSV ファイルを取り込む
- アプリは 2025-12-25 以降の履歴を投入する
- ユーザーは初期残高再現のために補正入力を行える

## 機能要件

### 実口座管理

- 銀行名、口座名、口座種別を登録できる
- 口座をアクティブ、非アクティブで管理できる
- 将来的にどの口座でも引き落とし管理対象にできる設計にする

### 目的別口座管理

- 目的別口座を作成、編集、停止できる
- 目的別口座は1つの実口座に属する
- 目的別口座の例
  - 親孝行
  - 定期支払
  - 特別貯金
  - 衣服・美容
  - 投資用
  - 趣味用
  - 健康用

### 取引管理

- 入金、出金、振替、口座間移動を記録できる
- 取引日、金額、対象口座、メモを保持できる
- 必要に応じて目的別口座との関連付けができる
- 残高計算の基礎データとして利用できる

### 引き落とし予定管理

- 引き落とし予定を手入力で登録できる
- 保持項目
  - 支払い名
  - 金額
  - 引き落とし日
  - 対象口座
  - 支払い種別
  - 繰り返し有無
- 対象口座ごとの引き落とし予定合計を確認できる
- 対象口座の残高と比較して不足の有無を判定できる

### CSV 取込

- 銀行明細 CSV を取り込める
- まずは対象フォーマットを限定してよい
- 2025-12-25 以降の明細取り込みを初期目標とする
- 取込後に補正や分類を手動で行える

### ダッシュボード

- 実口座ごとの現在残高を表示する
- 目的別口座ごとの現在残高を表示する
- 未配分残高を表示する
- 近い将来の引き落とし予定と不足見込みを表示する

## 非機能要件

- 個人利用前提で、まずはローカル環境とローカル本番環境で動作する
- monorepo 構成を採用する
- backend は Java + Spring Boot
- frontend は React + Vite + TypeScript
- DB は PostgreSQL
- Docker でローカル起動できる
- frontend の build 成果物は backend の static 配下に配置して配信する
- PC、スマホ、タブレットで利用しやすい UI にする
- iPhone、iPad から SSH 接続で作業できるよう、ローカル運用を意識する

## MVP 範囲

### Must

- 実口座の登録
- 目的別口座の登録
- 取引の手入力
- 目的別残高の表示
- 未配分残高の表示
- 引き落とし予定の手入力
- 対象口座の引き落とし不足判定
- CSV 取込の初版

### Should

- 口座間移動の扱いをわかりやすくする
- 月次の入出金集計
- 目的別口座への配分変更履歴

### Could

- 複数口座を横断した引き落とし余力表示
- CSV フォーマット追加対応
- カテゴリ分類の自動補助
- グラフ表示

### Won't for now

- 銀行 API 連携
- 自動同期
- マルチユーザー対応
- 本格的な家計簿分析機能

## 想定ドメインモデル

- `Account`
  - 実口座
- `GoalBucket`
  - 目的別口座
- `Transaction`
  - 入出金、振替、移動
- `ScheduledPayment`
  - 引き落とし予定
- `BalanceSnapshot`
  - 任意時点の残高確認用

## 将来拡張

- どの実口座でも引き落とし管理ができるようにする
- 取引 CSV から引き落とし候補や定期支払い候補を抽出する
- 複数年の履歴分析を行う
- 投資口座や資産クラス別の見える化を追加する
- 記事化しやすいように、開発ログや判断理由を docs に蓄積する

## 成功条件

- 現在の各実口座残高をアプリで把握できる
- 目的別残高を銀行の機能に依存せず維持できる
- 三菱UFJ銀行メイン口座の引き落とし不足を事前に把握できる
- 2025-12-25 以降の記録を継続して残せる
