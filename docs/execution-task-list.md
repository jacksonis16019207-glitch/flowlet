# flowlet 実行タスクリスト

## 目的

- 今の実装状況を前提に、次に着手すべきタスクを迷わず選べるようにする
- `Start.md` の目的である「目的別残高の見える化」「現在残高の再現」「引き落とし不足の事前確認」を優先する
- 実装、検証、記事化の区切りを作りやすくする

## 前提

- `Account`、`GoalBucket`、`Category / Subcategory`、`Transaction / Transfer / GoalBucketAllocation` の基本 CRUD と画面は着手済み
- CSV 取込は現時点のスコープに含めない
- 今後の優先順位は「プロダクト価値」「学習価値」「既存実装との接続しやすさ」で決める

## 優先順位

### P1. 残高集計を完成させる

目的:
実口座残高、GoalBucket 残高、未配分残高を一貫したルールで確認できる状態にする

実行タスク:
- ダッシュボードで必要な表示項目を確定する
- 残高集計 API のレスポンス案を決める
- `Account`、`GoalBucket`、未配分残高の計算ロジックを整理する
- バックエンドに残高集計 API を追加する
- フロントエンドにダッシュボード画面を追加する
- 残高計算ルールのテストを追加する

完了条件:
- 口座ごとの現在残高を画面で確認できる
- GoalBucket ごとの残高を画面で確認できる
- 未配分残高を画面で確認できる
- 取引、振替、配分を行った結果が集計へ反映される

### P2. 取引系のテストを補強する

目的:
既に入っている取引系機能を安全に拡張できる状態にする

実行タスク:
- `Transaction` の正常系と異常系を整理する
- `Transfer` の 2 明細生成をテストする
- `GoalBucketAllocation` の複数配分展開をテストする
- 削除時の関連データ削除をテストする
- 残高計算に影響する主要ケースをテストへ追加する

完了条件:
- 取引、振替、配分の主要ルールが自動テストで固定されている
- 回帰時に壊れた箇所をテストで検知できる

### P3. 開発用 seed データを整理する

目的:
画面確認、手動検証、記事用スクリーンショットの準備を速くする

実行タスク:
- 公開用に使えるダミーデータ方針を再確認する
- 固定マスタデータを `infra/sql/master-data/` に整理する
- 開発用データを `infra/sql/dev-seed/` に整理する
- 取引、振替、配分、残高確認がすぐできるデータセットを作る
- seed 投入手順を `docs/setup.md` に追記する

完了条件:
- ローカル環境で seed を投入すれば主要画面をすぐ確認できる
- 公開してよいデータだけで動作確認できる

### P4. ScheduledPayment を実装する

目的:
引き落とし不足の事前確認という主要ユースケースを成立させる

実行タスク:
- `ScheduledPayment` の責務を定義する
- DB テーブルと migration を追加する
- CRUD API を追加する
- 不足判定ロジックを追加する
- 一覧画面と登録フォームを追加する
- 不足見込み表示をダッシュボードまたは専用画面へ追加する
- 業務ルールのテストを追加する

完了条件:
- 引き落とし予定を登録できる
- 対象口座ごとの予定額合計を確認できる
- 不足見込みの有無を確認できる

### P5. UI の整理と操作性改善

目的:
今ある機能を日常的に使いやすい形へ整える

実行タスク:
- 取引画面の情報量を整理する
- フィルタ、並び順、表示グルーピングの必要性を確認する
- ダッシュボードから各画面への導線を整理する
- エラーメッセージと空状態表示を見直す
- モバイル幅での崩れを確認する

完了条件:
- 主要画面で次に押す操作が分かりやすい
- 一覧と登録フォームの往復がしやすい
- PC とスマホ幅の両方で最低限の操作が成立する

## 直近のおすすめ実行順

1. 残高集計 API の仕様を決める
2. 残高計算テストを追加する
3. ダッシュボード API を実装する
4. ダッシュボード画面を実装する
5. 開発用 seed データを整理する
6. ScheduledPayment の設計に入る

## 今回明示的にスコープ外とするもの

- CSV 取込
- CSV 取込履歴
- CSV プレビュー
- CSV 手動補正フロー

必要になった場合は、新規タスクとして別途追加判断する。
