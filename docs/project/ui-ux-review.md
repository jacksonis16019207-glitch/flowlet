# UI / UX Review

最終更新: 2026-04-10
関連 Issue: #139

## 目的

- 主要画面の UI / UX 課題を、実装に分解しやすい粒度で整理する
- 情報量、導線、操作頻度、モバイル表示の観点で優先度を付ける

## 監査対象

- ダッシュボード
- 口座
- カテゴリ
- 取引
- 設定

## 総評

- 画面の骨格、配色、カード設計、モーダル基盤はすでに揃っている
- 主要画面に情報が集まり始めたぶん、導線の優先順位と情報の主従が曖昧になっている
- 取引、口座、ダッシュボードは機能の厚みが出た一方で、主操作がやや埋もれている
- モバイル配慮の CSS はあるが、入力量の多い画面では操作負荷がまだ高い

## 優先度付きの改善テーマ

### P1. 取引画面の情報設計を整理する

- 1 画面に `transaction / transfer / allocation`、一括入力、クイック作成、明細、詳細が同居している
- 高頻度画面だが、初回理解コストと入力時の判断負荷が高い
- 「今なにをする画面か」を明確にし、主タスクと補助タスクを分離したい

改善観点:
- 主操作を 1 つに絞った初期状態
- 一括入力、振替、配分、カテゴリ追加の露出順の整理
- 明細一覧と詳細の関係の簡素化
- モバイル時の入力フロー短縮

### P1. 口座画面の一覧と詳細の切り替えを軽くする

- 一覧、絞り込み、詳細、関連 GoalBucket、関連カード、最近の取引までを 1 画面で担っている
- 情報としては有用だが、選択後の視線移動が長く、モバイルでは一覧と詳細の往復が重い
- 「一覧で見ること」と「1 件を深掘ること」をもう少し明確に分けたい

改善観点:
- 一覧のサマリー強化
- 詳細パネルの優先情報の絞り込み
- モバイル時の戻る導線と文脈保持
- クレジットカード口座と通常口座の差分表現

### P1. ダッシュボードの主従関係を整理する

- サマリー、Today Focus、口座、GoalBucket、月次収支、カテゴリ内訳がすべて同等に強く見える
- 情報量は十分だが、「最初に見るべき指標」が埋もれやすい
- 現在の意思決定に直結する情報を先頭でより強く見せたい

改善観点:
- 最上段に置く KPI の厳選
- `Today Focus` と下位セクションの役割分離
- 詳細リストを段階的に開く構成
- モバイル時のスクロール長削減

### P2. カテゴリ管理を管理画面として整理する

- カテゴリとサブカテゴリの編集・追加・展開が 1 つのカードの中に密集している
- 管理者向け画面としては十分だが、操作の可発見性と編集状態の見通しを上げたい
- 一覧性を保ちつつ、編集状態をもっと明確にしたい

改善観点:
- 閲覧モードと編集モードの区別
- 展開中カテゴリの視認性向上
- 新規追加導線の固定化
- 件数、状態、並び順の見せ方整理

### P2. 設定画面を「確認しながら変更する」画面に寄せる

- 現在は月初基準日の変更に機能が絞られている一方、説明と現在値表示が弱い
- 変更の影響が想像しづらく、確認体験としてはまだ薄い
- 設定値の意味と反映範囲を補足したい

改善観点:
- 現在値と変更値の比較
- この設定が影響する画面の明示
- 例示付き説明
- 将来の設定項目追加を見据えた構成

## 画面別メモ

### ダッシュボード

- ヒーロー、Focus、各詳細セクションの強さが近く、優先順位が見えにくい
- 一覧カードが多く、モバイルで縦に長くなりやすい

### 口座

- 絞り込みと一覧は十分だが、詳細パネルの責務が広い
- 信用口座と通常口座で必要な情報の差が大きい
- モバイルの一覧復帰導線はあるが、情報密度が高く往復負荷がある

### カテゴリ

- 管理対象を一箇所に集約していて便利
- ただしカード内の編集 UI が増え、一覧モードとしては少し重い

### 取引

- 機能は強いが、最も複雑
- 高頻度画面なので、機能量より操作速度を優先した再整理が必要

### 設定

- シンプルで迷いにくい
- その反面、変更前に理解を助ける補助情報が足りない

## 後続 Issue に分解する単位

- 取引画面の情報設計見直し
- 口座画面の一覧 / 詳細 / モバイル導線の再整理
- ダッシュボードの KPI 優先度とセクション構成の再設計
- カテゴリ管理画面の編集導線整理
- 設定画面の説明と確認導線の強化
