# flowlet MVPタスク

## 方針

最初のゴールは、`flowlet` の土台を作り、目的別残高の見える化と引き落とし不足判定をローカルで確認できる状態にすること。

## マイルストーン

### M1: monorepo の土台作成

- Git リポジトリを初期化する
- `backend/`, `frontend/`, `docs/` を作成する
- ルートに `README.md` を作成する
- 開発コマンドと構成方針を `docs/` に記載する

### M2: 開発環境の起動

- PostgreSQL の Docker 構成を作る
- Spring Boot バックエンドの雛形を作る
- React + Vite フロントエンドの雛形を作る
- backend と frontend の起動手順を統一する

### M3: バックエンドの最小機能

- 口座管理 API を作る
- 目的別口座管理 API を作る
- 取引管理 API を作る
- 引き落とし予定管理 API を作る
- 残高集計 API を作る

### M4: フロントエンドの最小画面

- ダッシュボード画面
- 実口座一覧画面
- 目的別口座一覧画面
- 取引入力画面
- 引き落とし予定画面

### M5: CSV 取込初版

- 対象 CSV フォーマットを1つ決める
- アップロードから取込までの流れを作る
- 取込結果の確認画面を作る
- 直近の給料日以降の履歴を投入できるようにする
