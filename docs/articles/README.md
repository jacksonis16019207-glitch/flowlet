# flowlet 記事管理

## 目的

`flowlet` の開発を、ポートフォリオだけでなく学習記録と発信にもつなげるため、記事ネタと下書き優先度を管理する。

## 運用方針

- 開発の節目ごとに記事候補を追加する
- 実装ログではなく、課題、設計判断、学びが伝わるテーマを優先する
- 1記事につき 1 論点を原則にする
- 公開順は、技術詳細よりも「なぜ作るか」が先になるようにする

## 公開候補一覧

### 優先度 A

1. 目的別口座の運用を見直したのをきっかけに、家計・資産管理アプリを作ることにした
2. flowlet の要件定義: 目的別残高の見える化と引き落とし不足判定をどう整理したか
3. Java 25 + Spring Boot 4 + React + Vite で家計・資産管理アプリの monorepo を立ち上げた

### 優先度 B

4. 実在する銀行口座とアプリ内の目的別口座を分ける設計
5. 家計簿アプリではなく、資金配置アプリとして flowlet を始めた理由
6. 個人開発で DDD をどう取り入れるか
7. AI コーディングエージェントを個人開発でどう使うか
8. 個人開発アプリを公開するとき、実データをどう避けるか: flowlet のダミーデータ方針

### 優先度 C

9. CSV 取込を MVP に入れる理由と設計方針
10. 引き落とし不足判定のロジックをどうモデル化するか
11. モバイルでも見やすい家計・資産管理 UI をどう組み立てるか
12. 残高集計の次に、月次収支をどう切り出したか
13. 家計簿ダッシュボードでカテゴリ別収支をどう見せるか
