# 公開用ダミーデータ方針

## 目的

`flowlet` をポートフォリオや記事で公開する際に、実在の口座情報や個人の資金移動を含めずに説明できる状態を保つ。

## 公開しない情報

- 実在の銀行名、支店名、口座名、口座番号
- 実際の残高、入出金額、引き落とし額、給与額
- 実際の日付、周期、振替ルール
- 個人を推測できるメモ、カテゴリ名、相手先名
- スクリーンショット内の URL、ID、通知、履歴などの周辺情報

## 公開用データのルール

- 銀行名は `メイン銀行` `決済用銀行` `貯蓄用銀行` のような汎称に置き換える
- 口座名は役割ベースで表現し、固有の運用名は使わない
- 金額は固定のサンプル値を使い、実額との対応関係を持たせない
- 日付は公開用の基準日にずらし、実生活のタイミングを再現しない
- 資金移動はユースケース説明に必要な最小本数だけ残す
- CSV、seed、画面表示、記事サンプルで同じ公開用データセットを使う

## このリポジトリでの運用

- `docs/project/requirements.md` は公開前提の表現を使い、固有名詞や実額を書かない
- DB seed は必ず架空データを使い、実口座の再現に使わない
- フロントエンドのモックやスクリーンショットも同じ架空データにそろえる
- 環境変数、ローカル CSV、個人メモは `.env` や追跡対象外ファイルで管理する
- ログやエラーメッセージに個人情報を出さない

## ダミーデータの命名例

- 銀行: `メイン銀行` `決済用銀行` `貯蓄用銀行`
- 口座: `生活口座` `固定費口座` `貯蓄口座`
- 目的別口座: `生活イベント` `定期支払` `特別費` `趣味` `健康` `投資`

## 公開前チェック

- 実在の銀行名やサービス名が残っていないか
- 実額と分かる金額や日付が残っていないか
- スクリーンショットに通知やタブ名が映っていないか
- seed / fixture / docs / README の表現が一致しているか
