# P5 取引入力を効率化する

更新日: 2026-04-06

## 完了メモ
- 取引詳細から `コピーして新規作成` を実行できるようにした
- 通常取引フォームに `単件入力 / 一括入力` の切り替えを追加した
- 一括入力では共通項目を固定したまま複数明細を連続登録できるようにした
- `前回入力を戻す` を追加し、直前の通常取引入力を再利用できるようにした

## 実装意図
- 日々の入力では、完全な新規作成よりも類似明細の再利用が多い
- 口座、カテゴリ、日付などの共通項目は維持し、明細だけを差し替えられる方が入力負荷を下げやすい
- 既存 API をそのまま使い、フロントエンド主導で P5 の価値を先に出す

## 記事候補
- 取引入力を「新規作成だけの画面」から「再利用中心の画面」へ改善した話
