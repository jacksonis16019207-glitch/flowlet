# Webアプリの作成

以下の条件をすべて満たしてWebアプリを作成したい。
どんなWebアプリにするかの要件定義などは土台ができた後で詰めようと思います。
まずは何をしたらいいですか？

## Webアプリ開発の目的

- ポートフォリオとして役立てたい
- 記事も出せたらいいと思っている (いままでやったことないし、何を記事にしていいかもわからないのでアドバイスも欲しい。開発を進める中でこれ記事にするといいみたいなことも教えてほしい。どこにどうやって記事を挙げるのかも教えて欲しい)
- Reactやtypescriptの勉強も一緒にしたい
- ドメイン駆動開発の勉強も一緒にしたい
- AIコーディングエージェントに関しても一緒に勉強したい

## 基本情報

- プロジェクト名はflowlet
- monorepoを考えている

## 技術

- backend : Java、SpringBoot
- frontend : React、Vite
- DB : PostgreSQL
- インフラ : Docker
- ソース管理 : GitとGitHub
- AIコーディングエージェント : Codex
- 自分しか使わないWebアプリなのでひとまずは開発環境と本番環境は共にローカルに構築する予定
- アプリのURLは`http://localhost:XXXX`とし、開発環境と本番環境でポートを分ける
- フロントエンドは、nginx配信でなくbackendのstaticにビルド成果物を置く形にしたい
- iPhoneやiPadからもSSH接続できるようにしたい
- UI/UXはPC、スマホ、タブレットに対応し、モダンなデザインにしたい

## リポジトリについて

- 要件定義や設計書などのドキュメントなどを格納するdocsディレクトリもあるといい

## AIコーディングエージェントについて

- AIコーディングエージェントにはタスク管理や、勉強したいところに直接関係しないところの実装などを担当してほしい
- Skillsを勉強して有効活用していきたい
- どんなSkillsがあるといいか、これをSkillsにするといいといった提案も開発を進める中でしてほしい
- コミットやプッシュ、プルリク、ドキュメント作成、要件定義、バックエンド実装やフロントエンド実装、記事作成などはSkillsにしたい

- どこにどうやって記事を挙げていいかわからないので、手取り足取り教えてほしい
- 開発を進める中でこれ記事にするといいといった提案をしてほしい

## 私の開発環境について

- Windows
- IDEにIntelliJやWebStrormなどのJetBrains製品はAll Productsを契約しているので自由に使える
- DBを触るツールとしてA5:SQLはインストール済み
- Docker Desktopはインストール済み
- GitHubアカウントあり
